# Contributing Guidelines

## How to Contribute

We welcome contributions to improve this project! Here's how you can help:

### Reporting Issues

- Check if the issue already exists
- Provide a clear description of the problem
- Include steps to reproduce if applicable
- Specify your Python version and environment

### Making Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Update documentation and requirements if needed
5. Commit with clear messages (`git commit -m 'Add feature description'`)
6. Push to your branch (`git push origin feature/your-feature`)
7. Create a Pull Request with a description of your changes

### Code Standards

- Write clear, readable code with comments where necessary
- Follow PEP 8 Python style guidelines
- Update the README if adding new features
- Test your changes before submitting a PR
- Keep notebooks clean and well-documented

### Pull Request Process

1. Update README.md with any new features or changes
2. Update requirements.txt if adding dependencies
3. Ensure all cells in notebooks run without errors
4. Add a clear description of your changes in the PR

## Questions?

Feel free to open an issue with the `question` label for any clarifications needed.

Thank you for contributing!
